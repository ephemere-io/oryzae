コアモデル
1. Entry（エントリ）
ユーザーが書く日記・テキストエントリ。IndexedDB（oryzae_board / entries）に永続化される。問いと N:M で紐づく。
カラム名型制約説明
idstringPK一意識別子（genId() で生成）
titlestringNOT NULLエントリのタイトル
toistring書き出し時の「問い」テキスト（ToiManager 経由）
contenttextエントリのプレーンテキスト本文
contentHTMLtextエントリの HTML 本文（Quill エディタ出力）
createdAtdatetimeNOT NULL作成日時（ISO 8601）
updatedAtdatetimeNOT NULL INDEX更新日時（一覧のソートキー）
writingModestring執筆時の縦書き/横書きモード
fontstring使用フォント名
metaobjectメタデータ（後述）
meta オブジェクトの主要フィールド： openedAt, darkMode, userFontSize, timeInscriptionEnabled, timeInscriptionMode, eraserTraceEnabled, erasureTraces, ghostEnabled, ghostMode, ghostSize, ghostScatter, ghostBlurStart, ghostBlurEnd, ghostDuration — および MetadataTracker が記録する執筆統計（WPM サンプル、文字数推移など）。
2. Question（問い）
ユーザーが設定する問い。最大3つまで。問いの文面は QuestionTransaction で版管理される。アーカイブ・Oryzae提案フラグを持つ。
カラム名型制約説明
idstringPK一意識別子（例: "q1"、新規は "q" + Date.now()）
user_idstringFK → User所有ユーザー
created_atdatetimeNOT NULL作成日時
updated_atdatetimeNOT NULL最終更新日時（アーカイブ・復活操作でも更新）
is_archivedbooleanDEFAULT falseアーカイブ済みかどうか。true の場合、jar 画面に非表示
is_validated_by_userbooleanDEFAULT trueユーザーが承認済みか。Oryzae 提案で未承認の場合 false
is_proposed_by_oryzaebooleanDEFAULT falseOryzae（発酵プロセス）によって提案された問いか
アクティブな問い = is_archived === false かつ is_validated_by_user !== false。
jar 画面・エディタドロップダウンには getActiveQuestions() でフィルタした問いのみ表示される。
3. QuestionTransaction（問いトランザクション）
問いの文面の版管理。ユーザー編集・発酵プロセスによる提案のたびに新バージョンが追加される。
カラム名型制約説明
idstringPK一意識別子（例: "qt1"、新規は "qt" + Date.now()）
question_idstringFK → Question対応する問い
stringtextNOT NULL問いの文面（最大64文字）
question_versionintegerUNIQUE (question_id, ver)バージョン番号（1始まり、単調増加）
created_atdatetimeNOT NULL作成日時
updated_atdatetimeNOT NULL更新日時
is_validated_by_userbooleanDEFAULT trueユーザーが承認済みか
is_proposed_by_oryzaebooleanDEFAULT falseOryzae による提案トランザクションか
最新文面の取得ロジック：getQuestionString() は is_validated_by_user !== false のトランザクションのうち最大バージョンの string を返す。
タイムライン表示：全バージョン履歴が問いの変遷タイムラインに時系列で可視化される。
4. EntryQuestionLink（エントリ-問い紐付け）
エントリと問いの多対多の関連付け。エディタ画面で問いをエントリにリンクする際に使用。
カラム名型制約説明
entry_idstringFK → Entry UNIQUE (entry, question)エントリの ID
question_idstringFK → Question問いの ID
発酵プロセスモデル
5. FermentationResult（発酵結果）
問いに対する一定期間の発酵分析結果。AnalysisWorksheet・Snippet・Letter・Keyword の親レコード。
カラム名型制約説明
idstringPK一意識別子（例: "fr1"）
question_idstringFK → Question対応する問い
target_periodstringNOT NULL分析対象期間（例: "2025-11-30〜2025-12-06"）
created_atdatetimeNOT NULL発酵完了日時
6. AnalysisWorksheet（分析ワークシート）
M-GTA に基づく概念分析。1回の発酵結果に複数の概念（ワークシート）が紐づく。UIには直接表示しないがデータとして保持。
カラム名型制約説明
idstringPK一意識別子（例: "aw1"）
fermentation_result_idstringFK → FermentationResult対応する発酵結果
concept_nametextNOT NULL概念名（例: "夢における一時的な邂逅と離別"）
definitiontextNOT NULL概念の定義
examplestext具体例（番号付きテキスト）
theoretical_memotext理論的メモ
7. CategoryRelation（カテゴリ関係）
複数の概念（AnalysisWorksheet）をまとめる上位カテゴリとストーリーライン。UIには直接表示しないがデータとして保持。
カラム名型制約説明
idstringPK一意識別子（例: "cat1"）
fermentation_result_idstringFK → FermentationResult対応する発酵結果
category_nametextNOT NULLカテゴリ名（例: "生死の境界の溶解"）
related_conceptsarray<string>関連する AnalysisWorksheet の ID 配列
storylinetextカテゴリのストーリーライン
8. ExtractedSnippet（抽出スニペット）
発酵プロセスで抽出されたユーザーのエントリからの引用断片。jar 画面の円内にカードとして表示。
カラム名型制約説明
idstringPK一意識別子（例: "snip1"）
fermentation_result_idstringFK → FermentationResult対応する発酵結果
snippet_typeenum種別: "new_perspective" / "deepen" / "core"
original_texttextNOT NULL抽出された原文テキスト
source_datestring出典日付（例: "12/4"）
selection_reasontext選出理由の説明
9. Letter（手紙）
Lab（分析AI）がユーザーに宛てて書く手紙。1回の発酵結果に対して1通。jar 画面で手紙アイコンとして表示。
カラム名型制約説明
idstringPK一意識別子（例: "letter1"）
fermentation_result_idstringFK → FermentationResult UNIQUE対応する発酵結果（1:1）
body_texttextNOT NULL手紙の本文
10. Keyword（キーワード）
Yeast（生成AI）がエントリから抽出・生成したキーワード。jar 画面の円内にタグとして表示。
カラム名型制約説明
idstringPK一意識別子（例: "kw1"）
fermentation_result_idstringFK → FermentationResult対応する発酵結果
keywordtextNOT NULLキーワード文字列（例: "沈黙の忠実さ"）
descriptiontextキーワードの説明文
永続化
