# @oryzae/shared パッケージガイド

## 概要

`packages/shared`（`@oryzae/shared`）は、モノレポ内のサーバーとクライアントで共有されるバリデーションスキーマと定数のパッケージである。Zod を依存に持ち、API リクエスト/レスポンスのバリデーションルールを単一ソースで管理する。

## 配置ルール

### 配置してよいもの

- **Zod スキーマ** — API リクエスト/レスポンスのバリデーション用
- **共有定数** — サーバーとクライアントの両方で参照される値

### 配置してはいけないもの

- **ドメインロジック** — ビジネスルールは `domain` 層に置く
- **ドメイン型**（`EntryProps` 等） — ドメインモデルは `domain` 層の管轄
- **インフラ層の関心事** — DB スキーマ、外部 API クライアント等
- **React コンポーネント** — UI はクライアント側に置く

## インポートルール

| 場所 | import 可否 | 理由 |
| --- | --- | --- |
| サーバー `domain` 層 | **禁止** | `@oryzae/shared` は zod に依存する。domain は外部依存ゼロの原則 |
| サーバー `application` 層 | **禁止** | application は domain のみに依存する |
| サーバー `presentation` 層 | **許可** | HTTP バウンダリでのリクエストバリデーションに使用 |
| クライアント `features/` | **許可** | フォームバリデーション、定数参照に使用 |
| クライアント `lib/` | **許可** | ユーティリティでの定数参照に使用 |

## 新規エクスポート追加時の注意

新しいスキーマや定数を追加する前に、以下を確認すること:

- サーバー `presentation` 層とクライアント `features/` の**両方で使われる**こと
- 片方でしか使われないものは共有パッケージに置く必要がない
- ドメインロジックが混入していないこと

## 現在のエクスポート一覧

| エクスポート名 | 種別 | 用途 |
| --- | --- | --- |
| `createEntrySchema` | Zod スキーマ | エントリ作成リクエストのバリデーション |
| `questionStringSchema` | Zod スキーマ | 質問文字列のバリデーション |

| `MAX_CONTENT_LENGTH` | 定数 | エントリ本文の最大文字数 |
