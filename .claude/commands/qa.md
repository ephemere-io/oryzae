# QA: クライアント UI テスト

Playwright MCP を使って、クライアントアプリの UI テストを実行する。
テスト前に `http://localhost:3001` が起動していることを確認し、起動していなければ起動する。
バックエンド `http://localhost:3000` も必要。

テストアカウント: `test@oryzae.dev` / `testpass123`

## テストケース

### 1. 認証フロー

#### 1-1. ログイン
1. `http://localhost:3001/login` に遷移
2. メールアドレス `test@oryzae.dev` を入力
3. パスワード `testpass123` を入力
4. 「ログイン」ボタンをクリック
5. **期待**: `/entries` にリダイレクトされる
6. **期待**: ヘッダーに「エントリ」「質問」「ログアウト」が表示される

#### 1-2. ログアウト
1. ログイン済み状態で「ログアウト」をクリック
2. **期待**: `/login` にリダイレクトされる

### 2. 問い一覧 (`/questions`)

#### 2-1. タイムライン表示
1. ログイン後、ヘッダーの「質問」をクリック
2. **期待**: 「問いの変遷」タイトルと「あなたの問いが生まれ、育ち、変化してきた記録」サブタイトルが表示される
3. **期待**: タイムラインにイベントカードが表示される（問いが存在する場合）
4. **期待**: 各カードに色付きドット + イベントラベル（作成/提案/アーカイブ）+ テキスト + 帰属表示がある

#### 2-2. 問いの作成
1. 「新しい問いを追加...」入力欄にテキスト（例: 「テスト問い」）を入力
2. 「追加」ボタンをクリック
3. **期待**: タイムラインに新しい問いが追加される
4. **期待**: 入力欄がクリアされる

#### 2-3. 問いのアーカイブ
1. アクティブな問いカードの「アーカイブ」ボタンをクリック
2. **期待**: カードのラベルが「アーカイブ」に変わる
3. **期待**: ボタンが「復元」に変わる

#### 2-4. 問いの復元
1. アーカイブ済みの問いカードの「復元」ボタンをクリック
2. **期待**: カードのラベルが「作成」に戻る
3. **期待**: ボタンが「アーカイブ」に戻る

### 3. エディタ (`/entries/new`)

#### 3-1. エディタ画面表示
1. `/entries/new` に遷移
2. **期待**: フルスクリーンエディタが表示される
3. **期待**: 上部ツールバー — 設定(⚙)、フルスクリーン(⛶)、日時表示、閉じる(×)
4. **期待**: 問い紐づけ — ドロップダウン + 丸い(+)ボタン
5. **期待**: テキストエリア — placeholder「今日のことを書いてみましょう...」
6. **期待**: 下部ツールバー — 新規(📄)、保存(⬆)、一覧(📁)、マイク(🎙, disabled)
7. **期待**: ステータスバー — 「● EDITING」左、プログレスバー中央、「0 CHARS」右

#### 3-2. テキスト入力 + 文字数カウント
1. テキストエリアに「テスト入力」と入力
2. **期待**: ステータスバーの文字数が「5 CHARS」に更新される
3. **期待**: プログレスバーが微量進む

#### 3-3. 問い紐づけ
1. ドロップダウンから問いを選択
2. (+) ボタンをクリック
3. **期待**: 問いタグがドロップダウン下に表示される
4. **期待**: ドロップダウンが「問いを紐づける…」に戻る
5. **期待**: 選択済みの問いがドロップダウンから消える

#### 3-4. 問いタグ解除
1. 紐づけ済みの問いタグの × をクリック
2. **期待**: タグが消える
3. **期待**: 問いがドロップダウンに再び表示される

#### 3-5. 保存（新規作成）
1. テキストを入力し、問いを紐づけた状態で保存ボタンをクリック
2. **期待**: URL が `/entries/{uuid}` に変わる
3. **期待**: テキストと問い紐づけが保持される

#### 3-6. 設定パネル
1. 設定(⚙)ボタンをクリック
2. **期待**: 文字サイズスライダーが表示される（14-36px）
3. スライダーを変更
4. **期待**: テキストエリアのフォントサイズが変わる

#### 3-7. 一覧に戻る
1. 右上の × ボタンをクリック
2. **期待**: `/entries` に遷移する

#### 3-8. 保存済みエントリの編集
1. `/entries` 一覧からエントリをクリック
2. **期待**: エディタが開き、既存のテキストと紐づけ済みの問いが表示される
3. テキストを変更して保存
4. **期待**: 変更が保存される

### 4. ナビゲーション

#### 4-1. ヘッダーナビ
1. 各ヘッダーリンク（エントリ / 質問）をクリック
2. **期待**: 対応するページに遷移する

#### 4-2. エディタ内ナビ
1. エディタ下部の「一覧」ボタンをクリック → `/entries` に遷移
2. エディタ下部の「新規エントリ」ボタンをクリック → `/entries/new` に遷移

## 実行方法

各テストケースを Playwright MCP のブラウザ操作ツールで実行する:
- `browser_navigate` でページ遷移
- `browser_fill_form` でフォーム入力
- `browser_click` でボタン操作
- `browser_snapshot` でDOM構造確認
- `browser_take_screenshot` でスクリーンショット取得
- `browser_console_messages` でエラー確認

各テストケースの実行後、結果を ✅ PASS / ❌ FAIL で報告する。
FAILの場合はスクリーンショットとエラー内容を添える。
