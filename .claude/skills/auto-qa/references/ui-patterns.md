# UI パターン参照

各画面の表示パターン。テスト時の期待値として使用。
実際の実装は `apps/client/src/features/` を動的に Read して確認すること。

## ログイン画面 (`/login`)

- タイトル「Oryzae」
- サブタイトル「ログインして続ける」
- メールアドレス入力欄
- パスワード入力欄
- 「ログイン」ボタン
- 「アカウントをお持ちでない方は サインアップ」リンク

## サインアップ画面 (`/signup`)

- タイトル「Oryzae」
- サブタイトル「アカウントを作成」
- メールアドレス入力欄
- パスワード入力欄
- 「サインアップ」ボタン
- 「すでにアカウントをお持ちの方は ログイン」リンク

## ヘッダー（認証済み）

- 「Oryzae」ロゴ（`/entries` へのリンク）
- 「エントリ」リンク
- 「質問」リンク
- 「ログアウト」ボタン

## エントリ一覧 (`/entries`)

- 「エントリ」見出し + 「新規作成」ボタン
- エントリカード（コンテンツプレビュー 120文字 + 日時）
- 「もっと見る」ボタン（20件以上ある場合）
- エントリなし: 「まだエントリがありません。最初のエントリを書いてみましょう。」

## 新規エントリ (`/entries/new`)

- 「新しいエントリ」見出し
- テキストエリア（placeholder「今日のことを書いてみましょう...」）
- 「キャンセル」ボタン + 「作成」ボタン

## エントリ編集 (`/entries/:id`)

- 「エントリ編集」見出し + 「削除」ボタン（赤）
- テキストエリア（既存の内容が入力済み）
- 「キャンセル」ボタン + 「更新」ボタン

## 質問管理 (`/questions`)

- 「質問」見出し
- 新規質問入力欄 + 「追加」ボタン
- セクション分け:
  - 「提案中」（amber 色、Oryzae 提案で未承認のもの）
  - 「アクティブ」
  - 「アーカイブ」
- 各質問カード: テキスト + アクション（アーカイブ/復元/承認/却下）
