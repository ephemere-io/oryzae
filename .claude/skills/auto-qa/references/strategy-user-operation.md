# Strategy B: ユーザー操作テスト

実際にページ遷移・クリック・フォーム入力を行い、ユーザー目線で期待通りの表示になるか確認する。

## いつ使うか

- エントリの作成・編集・削除が正しく動作するか確認したい
- 質問の作成・アーカイブ・提案承認が正しく動作するか確認したい
- ログイン/サインアップフローが正しいか確認したい
- ナビゲーション（ヘッダー、リダイレクト）が正しいか確認したい

## 手順

### 1. ページ遷移

```
navigate_page で対象ページへ:
- ログイン: http://localhost:3000/login
- サインアップ: http://localhost:3000/signup
- エントリ一覧: http://localhost:3000/entries
- 新規エントリ: http://localhost:3000/entries/new
- エントリ編集: http://localhost:3000/entries/{id}
- 質問管理: http://localhost:3000/questions
```

### 2. スナップショット取得と検証

```
1. take_snapshot で DOM 構造を取得
2. 期待するテキスト・要素が存在するか確認
3. take_screenshot で視覚的な確認（レイアウト崩れ等）
```

### 3. フォーム入力

```
- fill でテキスト入力（メール、パスワード、エントリ内容、質問テキスト）
- click でボタン操作（ログイン、保存、追加、アーカイブ等）
- wait_for で遷移・更新を待つ
```

## 機能別の確認ポイント

### 認証
- ログイン → `/entries` にリダイレクト
- サインアップ → `/entries` にリダイレクト
- ログアウト → `/login` にリダイレクト
- 未認証で `/entries` → `/login` にリダイレクト

### エントリ
- 一覧: エントリカードの表示（内容プレビュー、日時）
- 新規作成: テキスト入力 → 保存 → 一覧に反映
- 編集: 既存内容の表示 → 変更 → 保存
- 削除: 確認ダイアログ → 削除 → 一覧から消える
- ページネーション: 「もっと見る」ボタン

### 質問
- 一覧: アクティブ/提案中/アーカイブのセクション分け
- 作成: テキスト入力 → 「追加」→ 一覧に反映
- アーカイブ/復元: ボタンクリック → セクション移動
- 提案の承認/却下: ボタンクリック → セクション移動

## Tips

- `wait_for` でテキストの出現を待ってからスナップショットを取ると確実
- `take_screenshot` は視覚的な確認に有効（特にレイアウト崩れ）
- フォーム送信後は `wait_for` でリダイレクト先の要素を待つ
